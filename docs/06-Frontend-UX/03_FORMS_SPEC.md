# Forms Spec

## Общие требования

- React Hook Form + Zod.
- Все обязательные поля отмечены, но без визуального мусора.
- Ошибки понятным русским текстом.
- На mobile submit sticky внизу.
- Не использовать длинные таблицы для ввода.

## Форма нового ремонта

Поля:

### Клиент

- Имя клиента — required.
- Телефон — optional, но показывать маску/подсказку.
- Email — optional.

### Инструмент

- Тип — guitar/bass/other.
- Бренд — optional.
- Модель — optional.
- Серийный номер — optional.
- Заметка — optional.

### Ремонт

- Описание ремонта — required.
- Цена ремонта — required, money input.
- Дата приема — default today.
- Ответственный мастер — optional/required по настройке.
- Комментарий — optional.

После сохранения:

- показать success toast;
- перейти на detail screen;
- dashboard invalidated.

## Форма оплаты

Поля:

- Сумма — default remaining amount.
- Способ оплаты.
- Дата оплаты — default now/today.
- Комментарий.

Показать сверху:

- цена заказа;
- уже оплачено;
- осталось оплатить.

## Форма расхода

Поля:

- Дата — default today.
- Категория.
- Сумма.
- Кто потратил.
- Способ оплаты.
- Описание.
- Связанный заказ — searchable optional.
- Комментарий.

## Форма фактической комиссии

Поля:

- Автоматическая комиссия — read-only.
- Фактическая комиссия — money input.
- Причина изменения — required.

При сохранении audit log обязателен.
